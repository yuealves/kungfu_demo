## 2. 当前默认合成逻辑

脚本当前默认参数等价于:

- `boundary_mode=jq_doc`
- `include_call_auction_first_bar=True`
- `price_tick_mode=trade`
- `ignore_nonpositive_current=True`
- `carry_session_end_tick=True`
- `extrema_mode=hybrid_updates`
- `jq_like=True`

也就是说，默认就是“聚宽口径”。

### 2.1 分钟标签和切片规则

- 分钟标签使用“分钟结束时刻”
  - `09:31:00` 表示 `09:30:00` 到 `09:30:59.xxx`
  - `13:01:00` 表示 `13:00:00` 到 `13:00:59.xxx`
- 对于恰好落在整分钟的 tick:
  - 普通分钟边界上的 `HH:MM:00` 归到下一根 bar
  - `11:30:00` 和 `15:00:00` 保留在本分钟

### 2.2 开盘第一根 bar

- `09:25:00` 到 `09:29:59.xxx` 的集合竞价 tick 会并入 `09:31`
- 但价格合成时不会直接使用所有集合竞价快照
- 价格只使用“有成交变化”的有效 tick，且过滤 `current<=0`

### 2.3 盘中价格怎么合成

`open/close/high/low` 的基础逻辑是:

- 只保留“相对上一条 tick，`volume` 或 `money` 发生变化”的 tick
- 过滤 `current<=0`
- `open` = 分钟内第一条有效价格 tick 的 `current`
- `close` = 分钟内最后一条有效价格 tick 的 `current`
- `high/low` 基础上先用分钟内有效价格 tick 的 `current` 聚合

这里不用整分钟 `max(tick.high)` / `min(tick.low)` 直接生成分钟 `high/low`，因为 `tick_l1` 是快照，不是逐笔成交。快照里的 `high/low` 更像“截至当前的日内高低点”，直接聚合会把更早时刻的日内极值错误带进当前分钟。

### 2.4 分钟 high/low 的补充规则

在 `current` 聚合的基础上，脚本还额外吸收“本分钟内 `tick.high/tick.low` 的更新值”:

- 如果某分钟内 `tick.high` 相对上一条快照发生上调，则该更新值可参与分钟 `high`
- 如果某分钟内 `tick.low` 相对上一条快照发生下调，则该更新值可参与分钟 `low`

这条规则不是直接拿整分钟所有 `tick.high/tick.low` 聚合，而是只吸收“本分钟内发生更新”的部分。实证上，这一条对齐了绝大多数盘中剩余的 `high/low` 差异。

### 2.5 volume 和 money 怎么合成

- `tick.volume` / `tick.money` 视为日内累计值
- 分钟的 `cum_volume` / `cum_money` 取该分钟最后一条 tick 的累计值
- 分钟 `volume` = 本分钟累计量 - 上一分钟累计量
- 分钟 `money` = 本分钟累计额 - 上一分钟累计额

最终输出时:

- `money` 按聚宽风格截断为整数金额
  - 具体做法是 `floor(money + 1e-6)`
  - `1e-6` 只用于消掉浮点尾差，例如 `256927.999999`

### 2.6 空分钟怎么补

如果某一分钟没有有效成交价格 tick:

- `close` 用上一有效价格延续
- `open/high/low` 也填成该价格
- `volume=0`
- `money=0`

### 2.7 午盘和收盘边界

脚本会把 session 结束后的第一条 tick 回补到结束分钟:

- 午盘: 把 `11:30:00` 后、`13:00:00` 前第一条 tick 回补进 `11:30`
- 收盘: 把 `15:00:00` 后、`15:05:00` 前第一条 tick 回补进 `15:00`

这一步是为了贴近聚宽在边界分钟的切片方式。

### 2.8 收盘 `15:00` 的特殊处理

`15:00` 不能按普通分钟 bar 处理，当前脚本把它视为“收盘撮合 bar”:

- `open` = 收盘撮合前最后一个有效价
- `close` = 收盘撮合后的第一条有效价
- `high` = `max(open, close)`
- `low` = `min(open, close)`
- `volume` = 撮合后累计量 - 撮合前累计量
- `money` = 撮合后累计额 - 撮合前累计额

如果 `15:00:00` 恰好出现价格变化但量额未变，脚本也允许这条 tick 成为最后一根的 `close` 参考价。

### 2.9 开盘早段全 0 的异常处理

如果当天开盘初段 tick 全是 0，且该分钟没有成交:

- 不使用 0 填价格
- 用前一交易日最后一根 `1m close` 填 `open/high/low/close`
- `volume=0`
- `money=0`

这条规则解决了 `600265.XSHG` 这类案例。

### 2.10 全天没有任何 tick 的股票

如果某只股票当天在 `tick_l1` 中完全不存在，但聚宽 `price_1m` 仍给出了整天分钟线:

- 脚本用前一交易日最后一根 `1m close` 生成全天 240 根静态 bar
- `open=close=high=low=prev_close`
- `volume=0`
- `money=0`

这条规则解决了 `000711.XSHE`、`002231.XSHE` 等整天无 tick 但 `price_1m` 有数据的股票。

### 2.11 输出数值规范

为和聚宽 parquet 内的数值表示保持一致，脚本最终会把:

- `open/close/high/low` 转成 `float32`
- `money` 转成整数金额的 `float64`
- `volume` 转成 `float64`

这一步解决了之前大量“只有 `1e-5` 量级”的浮点表示差异。

## 4. 几个关键样本

### 4.1 `600265.XSHG`: 开盘前几分钟 tick 全 0

当天 tick:

- `09:24:35` 到 `09:34:14`
  - `current=0`
  - `high=0`
  - `low=0`
  - `volume=0`
  - `money=0`
- 第一条有效 tick 出现在 `09:34:29`
  - `current=19.60`
  - `volume=100`
  - `money=1960`

前一交易日 `2026-03-16` 最后一根 `1m close`:

- `19.73`

聚宽 `2026-03-17` 的 `09:31` 到 `09:34`:

- `open=close=high=low=19.73`
- `volume=0`
- `money=0`

所以这里聚宽显然不是从当天 tick 直接合成的，而是用前一交易日最后一根 `1m close` 做了回填。脚本现在已按这一规则处理。

### 4.2 `000711.XSHE`: 当天完全没有 tick

这只股票在 `2026-03-17` 的 `tick_l1` 中完全不存在，但聚宽 `price_1m` 有 240 根 bar，且全部相同:

- `open=close=high=low=4.65`
- `volume=0`
- `money=0`

而前一交易日最后一根 `1m close` 正好也是 `4.65`。

所以这里可以确认: 对“全天没有 tick”的股票，聚宽也会用前一交易日收盘参考价补整天静态分钟线。脚本现在已按这一规则处理。

### 4.3 `300750.XSHE`: 收盘撮合 bar

收盘前后 tick:

- `14:59:51` 价格 `406.41`
- `15:00:00` 价格 `405.00`

聚宽 `15:00`:

- `open/high=406.41`
- `close/low=405.00`

这说明 `15:00` 更像“收盘撮合 bar”，而不是普通的 `[14:59:00, 15:00:00)` bar。脚本现在已按这一规则处理。

## 5. 当前对聚宽 1m 合成规则的推断

基于这次数据验证，当前最合理的推断是:

1. 聚宽 1m 基础上是由 `tick_l1` 快照合成的。
2. 分钟价格并不是简单使用“该分钟所有快照”，而是更接近“有成交变化的有效快照”。
3. 分钟 `high/low` 以 `tick.current` 为主，同时会吸收分钟内 `tick.high/tick.low` 的更新。
4. `volume/money` 来自 tick 累计量额做差。
5. `11:30` 和 `15:00` 存在边界回补规则。
6. `15:00` 实际上是一个收盘撮合 bar。
7. 对开盘初段全 0 或全天无 tick 的异常票，会退回到“前一交易日最后一根 1m close”进行静态填充。
8. 聚宽最终落盘的数据类型和取整方式也会影响看起来是否“完全一致”。

## 6. 使用说明

默认运行:

```bash
python 1min_kbar/compare_jq_1m.py
```

如果你只是想“从 tick 直接生成聚宽口径的 1m 数据”，推荐使用新的精简脚本:

```bash
python 1min_kbar/generate_1m_kbar.py
```

默认情况下，它会读取:

- `jq_data/tick_l1/{date}.parquet`
- `jq_data/price_1m/{date}.parquet`
- `jq_data/price_1m` 目录下前一交易日的 `1m` 文件

并输出:

- `1min_kbar/generated_1m_{date}.parquet`

其中:

- 当天 `price_1m/{date}.parquet` 主要用于提供“当天股票全集”
- 前一交易日 `price_1m` 主要用于提供“前收回填”

如果你后面脱离这份样例数据来跑，并且手里没有当天 `price_1m`，也可以继续生成，但要注意:

- 如果不给 `--universe-path`
- 脚本只能保证输出 `tick` 里实际出现过的股票
- 对于“当天完全没有任何 tick，但聚宽仍会生成整天静态 bar”的股票，必须额外提供一个股票全集来源

这种情况下建议显式传:

```bash
python 1min_kbar/generate_1m_kbar.py \
  --tick-path /path/to/tick.parquet \
  --prev-1m-dir /path/to/price_1m \
  --universe-path /path/to/universe_or_today_1m.parquet \
  --output-path /path/to/generated_1m.parquet
```

限制股票:

```bash
python 1min_kbar/compare_jq_1m.py --code 600265.XSHG 000711.XSHE
```

自定义输出目录:

```bash
python 1min_kbar/compare_jq_1m.py --output-dir 1min_kbar/output_custom
```

## 7. `generate_1m_kbar.py` 使用说明

这个脚本是当前推荐使用的“纯生成版”脚本，只负责:

- 读取 `tick_l1`
- 按聚宽口径合成 1m
- 输出 parquet

默认运行:

```bash
python 1min_kbar/generate_1m_kbar.py
```

默认前提:

- 当日 tick 从 `1min_kbar/jq_data/tick_l1/{date}.parquet` 读取
- 前一交易日 1m 从 `1min_kbar/jq_data/price_1m` 读取
- 当天股票池默认优先使用 `1min_kbar/jq_data/price_1m/{date}.parquet` 里的 `code`
  - 这样可以覆盖“当天完全没有 tick，但聚宽仍输出静态 bar”的股票

常用示例:

```bash
python 1min_kbar/generate_1m_kbar.py --date 2026-03-17
```

```bash
python 1min_kbar/generate_1m_kbar.py --date 2026-03-17 --code 000001.XSHE 600000.XSHG
```

```bash
python 1min_kbar/generate_1m_kbar.py --date 2026-03-17 --output-path 1min_kbar/generated_1m_2026-03-17.parquet
```

如果你没有当天 `price_1m` 文件，但仍想生成分钟线，也可以额外指定股票池文件:

```bash
python 1min_kbar/generate_1m_kbar.py --date 2026-03-17 --universe-path /path/to/universe.parquet
```

这里的 `universe.parquet` 只需要有一列 `code` 即可。
